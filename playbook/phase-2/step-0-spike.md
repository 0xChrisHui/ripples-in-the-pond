# Phase 2 Step 0 — 合奏技术验证（Gate）

> 🎯 **目标**：验证合奏核心体验在 Next.js 里可行，通过后才分线开发
>
> **串行执行**，不分线。通过标准必须全部达成。

---

## 🎯 通过标准（全部达成才放行）

1. **真音频文件加载并播放** — 从 `public/sounds/` 加载一个 mp3，按键后 < 50ms 发声
2. **背景原曲 + 按键音效同时播放** — 两个音源叠加不冲突、不爆音
3. **录制 10 秒并回放** — 回放时按键音效的时间点与录制时一致（误差 < 100ms）
4. **390px 宽度下有可操作入口** — 手机上显示"请使用电脑键盘体验合奏"提示，不是空白页

## 📦 范围

- `src/spike/JamSpike.tsx`（临时，验证后删）
- `app/spike/page.tsx`（临时路由，验证后删）
- `public/sounds/test-kick.mp3`（一个短音效文件，验证用）
- 不装新包，纯 Web Audio API + CSS

## 🚫 禁止

- 不写正式组件
- 不改现有文件
- 不调 API

## 🔍 验证步骤

1. 浏览器打开 /spike
2. 页面自动播放背景音乐（点击页面激活 AudioContext 后）
3. 按 A-Z 键 → 听到音效叠加在背景上
4. 点击"录制" → 演奏 10 秒 → 点击"停止"
5. 点击"回放" → 听到刚才的演奏，节奏一致
6. 把浏览器窗口缩到 390px 宽 → 看到提示文字，不是空白

## 决策点

**通过** → 删除 spike 文件，分线开发 Track A/B。

**部分不通过**：
- 音频延迟 > 100ms → 减少同时发声数（最多 4 个），用更短的音效文件
- 回放不对齐 → 改用 `performance.now()` 替代 `Date.now()` 做时间基准
- 降级后重新验证

**严重不通过**（背景混音崩溃 / AudioContext 无法解锁）→ 暂停 Phase 2，回到 Phase 1 加固。
