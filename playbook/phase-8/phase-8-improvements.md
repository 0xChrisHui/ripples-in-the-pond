## R1 — 水面深度缩放（水位联动）

**一句话**:滚轮/双指调水位时,水面贴图随水位整体缩放,营造"水面离我们更近/更远"的纵深感;**球与镜头距离恒定**,只有水层推拉。

**核心行为**
- 水位升 → 水面贴图**放大**(涟漪变大、水感扑面、边缘溢出屏幕)。
- 水位降 → 水面贴图**缩小**(涟漪变小变密、露出更多水面)。
- 缩放连续跟水位 + 缓动(复用现有水位 lerp);**正在扩散的涟漪也随整张贴图一起缩放**。

**约束(红线)**
- 球不缩放、球与镜头距离恒定(只水层动)。
- 只缩放、**不虚化/景深**(塘浅 ~1m,纵深小)。
- 缩小不露边 → 水面图案平铺(repeat)延展。

**参数(默认+可调)**:幅度做小(浅塘),水位 0→1 ≈ 0.9×→1.2×(进参数板);锚点=画面中心。

**实现要点**:合成 pass 按水位缩放高度场采样(绕中心)→ 自动带动所有涟漪(含进行中);等于落地占位未做的「缩放」滚轮模式,但**只缩水层、不缩球**;现有「水位/缩放」互斥模式 → 合并为"水位即缩放"。

**验收**:升=涟漪整体放大+边缘溢出+球大小不变;降=涟漪缩小变密+露出更多水面+球大小不变+无露边;进行中的涟漪也缩放;连续跟手。

**待定**:缩放幅度最终手感(参数板);"露出更多"范围上限。

## R2 — 涟漪改正圆（现为椭圆）

**现象**:涟漪是椭圆,不是正圆。
**根因**:高度场 256×256(方形),屏幕是宽屏;滴水/涟漪用 UV(0–1 归一)距离算,UV 圆在宽屏上被横向拉伸成椭圆。
**方向**:涟漪距离计算按屏幕宽高比(aspect)校正(UV 乘 aspect,使距离在统一度量下) → 正圆。
**状态**:记录,待做(改动小)。

## R3 — 鼠标划水更丝滑（现像高频点击）

**现象**:鼠标划过水面像高频快速点击(一串离散"咚咚"),不连续。
**根因**:pointermove(节流 16ms)每次在指针处投一滴离散水滴,快速划 = 一串孤立点。
**两种思路**
- **调参**:降「滴水·移动」强度(更柔)+ 调小滴水半径;本质仍离散,快划仍有颗粒。
- **技术**(治本):① **路径插值**——上帧↔当前指针位置之间按距离补几个小滴,点连成线(**推荐**,改动小);② **线段盖印**——往高度场盖一条线(连续扰动);③ **按位移触发 + 强度随速度**,免时间节流疏密不均。
**状态**:记录,待做。

## R4 — 浮沉可见性不一致 +（标签闪烁但球不动）【bug】

**现象**:某球右下角数字标签闪烁;但**把水位调到该球处、滚轮上下触水、等待,球本身并没有可见的沉浮**——只有标签在闪。

**根因(已核代码,确认)**:浮沉(displayZ)的可见性在三个消费方**不一致**:
- 球的没入淡出 `SphereInstances` 走 `waterOn={flags.water}`(只认旧「水面」,**不认「扭曲水面」waterFx**)→ 用户开的是 waterFx、旧水面没开 → 球 dim **完全不随 displayZ 变**(`PondGL.tsx:151` + `SphereInstances.tsx:127`)。
- 标题 `SphereOverlay` 走 `waterOn={water || waterFx}`(`page.tsx:82`)→ **会**随 displayZ 振荡 → **只标题闪**。
- 球在 waterFx 下的浮沉本应是 **清晰↔被扭**(WaterDistort 用 displayZ),但默认折射很轻 → 太微弱、读不出"在沉浮"。

**修复方向**
1. **统一浮沉可见性**:让球的浮沉表现在 waterFx 下也明确(SphereInstances 认 waterFx,或专门给一个 waterFx 下的浮沉视觉),三个消费方对齐。
2. **消除标题闪烁**:标题不透明度对 sub 的响应加阻尼/平滑,或更陡的淡出阈值,别随小幅浮沉脉动。
3. **根本**:让"浮沉"在球上有**清晰可见**的表现(不只是微弱折射)——与"深度怎么可视化"(R1 同源)挂钩;可能需要更明确的深度线索(如随深度的轻微缩放/亮度,待定)。

**状态**:**确认为 bug**(浮沉可见性不一致 + 标题闪烁),待修(排期/timing 待用户定)。

## R5 — 深度调制效果（折射/月光/涟漪按层级 L 渐变）

**现状**:折射近二元(水上清晰/水下扭),无"多深"梯度。
**方案(已与用户对齐)**:统一一个**深度因子** `d = clamp((水位 w − 球深 z) / 塘深, 0, 1)`(0=贴水面、1=塘底,用**动态 displayZ**),三个效果各一条曲线:
- 折射 `refract(d) = R · d^a`(近轻深重)+ clamp 上限(别把深球扭糊)
- 月光 `moon(d) = M · (1−d)^b`(近强深弱)
- 涟漪响应 `ripple(d) = (1−d)^c`(近灵敏深钝)
曲线指数/系数(a/b/c)+ R/M 上限 = 参数板可调。
**红线**:不染色/不压黑/不缩放/不虚化 → 深度差异只走 折射+月光+涟漪。
**好处**:一个 d 驱动全部 → 顺手**统一 R4 的不一致** + 让浮沉"看得见"(浮沉时 d 连续变 → 三效果一起连续变);纯函数极便宜;是 **H5 公式契约的视觉姐妹**(`g(depth)`)。

**(2026-06-16 细化)三层深度模型**:用户指出"浮出水面的球应**悬在半空**、而非嵌在水里"。故按球相对水面的**带符号位置** `s = 球深 z − 水位 w` 分三带:
- **空中(s>0,浮出水面)**:球清晰、悬空;无折射、无漩涡;**投影到水面**(见 R8)。
- **贴面(s≈0)**:接触/破水——**局部裹挟漩涡(参考 #3)在此带最强**,向空中和水下都衰减。
- **水下(s<0)**:折射/月光/涟漪按深度 `d = clamp(−s/塘深, 0, 1)` 渐变(上面三曲线)。
即:per-object 漩涡是"贴面接触"效果、不是"一直嵌水里";d 函数只管水下那带。

**状态**:方向 + 三层模型已对齐,待排期实现。

## 参考资料提炼:`references/flower-water-ripples`（2026-06-16 review）

> 同族 WebGL 涟漪 demo(折射静态 water.jpg,日光明亮风);以下 6 点供借鉴进 R1/R3/R5 + 手感定稿。讨论中。

1. **丰富水面光照(最大可借鉴)**:漫反射 `light=(grad.x+grad.y)*2.4`(坡面提亮)+ 波峰高光 `spec=max(0,light-阈)*5.5` + **流动光**(飘移对角光带 sin + 游走柔光池/caustic)→ 静止时水也"活"。我们只有月光高光,缺漫反射 + 流动光池 → 借鉴(改月光冷白版)大幅提升"水感"。
2. **梯度折射 `puv += grad*uRefr`**:与我们 R3 重做一致 → 印证 R3 方向对。
3. **每对象局部漩涡(per-flower swirl)**:每朵花周围局部"旋转+位移+缩放"扰动区 → 我们球可借:每球一圈局部水流裹挟,更"嵌"在水里 + 喂 R5。
4. **CPU 粗网格 sim → 小纹理(替代 GPU ping-pong)**:sim 跑 CPU 160×N 粗网格(波动方程+阻尼)打包 8-bit 纹理上传,shader Linear 采样天然平滑。优:无 FBO/half-float 复杂度、全平台稳、避开"色斑";代价:CPU 算(很小)。→ 讨论是否替换我们的水 sim。
5. **指针轨迹按距离触发小滴**:移动 >2px 落小滴(按距离而非时间节流)→ 轨迹更密更顺。支持 R3「路径插值」。
6. **常驻微波(ambient kisses)**:每 0.4–1.7s 随机极轻小滴(强度 0.14)→ 印证 H4,但更轻更稀,可作调参参考。
（次要:2D canvas 粒子层叠 GL 做 sparkle/花瓣、cover 裁剪适配、保存/分享。）
