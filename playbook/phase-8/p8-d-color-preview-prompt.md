## 二、要模拟的场景

这是一个音乐 NFT 网站首页的配色预览。场景 = 全屏暗色"夜晚水塘"，水面漂浮着可点击的"音乐水珠球"。**目的是比颜色，不是比特效保真**——动效做到能看出每个颜色角色在动态中的样子即可。

必含元素（每个元素对应一个颜色角色，缺一个该主题就没法被完整评估）：

1. **背景**：该主题的三段渐变铺满全屏（从上到下或径向），按各主题"特殊元素"再叠对应光斑/水路/vignette。
2. **36 个水珠球**：半径 14–28px，伪随机散布（用固定种子的确定性伪随机，保证 11 套主题下布局完全相同，便于对比）；颜色从该主题 8 色板分配，标注"限 N 球"的颜色严格按限额。
3. **球的水珠质感**（纯 CSS/SVG 渐变）：中心较透、边缘较深的径向渐变 + 左上小高光 + 底部细弱内反光 + 1px 微亮边缘——读作"半透明水珠"而非发光星球。
4. **动效**（全部 transform/opacity）：
   - 球慢漂浮 bobbing：上下 4–8px，3–6s，36 球相位/周期各不相同；
   - 每球周期涟漪圈：描边圆从球身扩散淡出（stroke = token **ripple**），timing `cubic-bezier(0.2, 0.6, 0.35, 1)`，周期 7–12s 错峰；
   - 点击空白处：从点击点荡开一圈大涟漪（stroke = token **light**）；
   - 点击球：球短暂变亮 + 球心荡开涟漪（token **light**）——模拟"播放"反馈；
   - 画面下部一层雾：自下而上的渐变（token **mist**，低透明度）；
   - 上 1/3"月光带"：约 20 条细短横线碎光原地闪烁（token **glow**）+ 3–4 个缓慢游走、呼吸明灭的浮光点（token **accent**）。
5. **仿站点顶栏**：左上角细体字 `Ripples in the Pond`（letter-spacing 0.3em，白色 80% 透明度）——用于判断各背景下顶栏可读性。

## 四、预览器自身的交互要求

1. **主题切换条**（底部常驻）：11 个按钮（`A 墨绿深潭` … `K 热带泻湖`），高亮当前项；键盘 ←/→ 循环切换。
2. **信息面板**（可收起）：当前主题名 + 球色板 8 个 swatch + token 5 个 swatch（标注角色名），便于对照。
3. **全览模式**：一个按钮切到 11 宫格静态缩略卡（每卡 = 该主题背景渐变 + 一排色板小球 + 主题名），用于先粗筛，点卡片进入该主题全屏。
4. **对比度检查开关**：开启后每球旁标注其填充色对背景基色的 WCAG 对比度数值，**< 3.0 标红**（球是可点击物，3:1 是硬红线）。
5. **主题切换实现**：CSS 变量 + 球重新着色，球的布局/大小/动画相位完全不变（只换颜色，便于公平对比）；切换带约 0.4s 颜色过渡。
6. 加分项（非必须）：A/B 并排分屏对比两套主题。

## 五、技术约束

- 单 HTML 文件，原生 JS，无任何外部资源 / CDN / 字体 / 图片。
- 动画只用 transform / opacity，保持流畅即可，不必工程级优化。
- 一次性输出完整文件；如长度受限，优先保证 11 套数据完整和切换功能，砍加分项。

## 六、我评估时会看什么（请按此对齐效果优先级）

1. 球（可点击物）在该背景上是否显著、暖冷层次是否成立；
2. token 五色角色是否分明：涟漪/高光/雾/碎光各司其职，accent 是该主题唯一的高显色点缀；
3. 顶栏白字在该背景上是否可读；
4. 整体气质是否对得起主题命名（如"墨绿深潭"应禅静、"雨夜霓虹"应赛博）。
