# P8-S1 视觉方向调研 — 三套水塘设计方案

> **产出自**：`visual-research-prompt.md`（主视觉全量快照提示词）+ 3 路并行网络调研（色彩方向 / 纯 SVG 技术 / 动效语言）。
> **日期**：2026-06-12。**状态**：等用户拍板方向。
> **口径**（用户 2026-06-12 更新）：弱化宇宙感**和 3D 感**；球体质感 = 池塘中的典型事物（不限于水珠）。
> **抽象口径**（2026-06-12 二次更新）：氛围元素**禁止具象形态**（鱼/水黾/落叶/莲叶等一律不做），全部抽象为光、波、斑、雾、痕；本文档中的具象映射（水黾/游鱼/落叶/莲叶）仅作运动节奏与角色的意象参考，抽象版落地规格见 `phase-8-b-effects-migration.md` §2.8-2.12。所有新效果先在 /test 页做成独立开关供用户体验。

## 〇、调研最重要的三个发现

1. **水感的 80% 不靠渲染精度**：获奖水主题网站里 WebGL 只负责折射精度；水感真正来自——交互因果（点击→涟漪）、运动节奏（慢、减速扩散）、统计随机（错峰、尺寸分布）。这三样纯 SVG 全能做，我们已有一半（`bg-ripple:wave` 总线、hash 错峰）。
2. **最便宜的"大杀器"是椭圆透视涟漪**：把涟漪 circle 压成 ellipse（ry≈rx×0.35），视角立刻从"俯视雷达波"变成"斜视水面"，零新增成本。这一招同时服务"弱化 3D 感"——水面本来就近乎平面。
3. **P8-A 滤镜路线可行但有三条铁纪律**：噪声静止（绝不逐帧动 baseFrequency）、36 球共享滤镜 def 且按 z 分 2–3 组 `<g>` 挂载（顺便实现"水深=晃动幅度"）、displacement scale 只做事件脉冲（`bg-ripple:wave` 到达时 14→26→14）。

## 一、三套设计方向

### 方向 A：墨绿深潭（Zen Ink Pond）★ 主推

**气质**：莫奈《绿色倒影》× 日式苔庭。最贴 overview 的"深青/墨绿"题中之义，离现有 Portra 暖球最近、迁移成本最低，靠青橙互补天然保住球的显著性（"水面调青、小球保暖"）。

- **背景**：`#05100e`（近黑墨绿，明度与现 `#07070f` 几乎相同 → 雾/景深/对比参数全套沿用）。水面渐变 `#05100e → #0a1f1a → #0e2a24`，offset 偏上的月光斑 radialGradient `#14342c` @8%。
- **球色板（8 色示范）**：`#D8A878`（暖杏·保留）、`#B8684A`（柿锈）、`#C9899B`（睡莲粉）、`#E8E2C8`（月光米）、`#7EA898`（池水绿·保留）、`#5C8A6A`（荷叶绿）、`#2E7066`（深青）、`#4A7878`（青石板）——4 暖 4 冷，暖进冷退自带层次。
- **光色**：高光月光象牙 `#EDE6CE` / 强 `#F6F0DC`；涟漪描边白 → `#C8D8CC`。
- **参考关键词**：`Monet green reflections nympheas Orangerie`、`zen pond moss garden night`、`dark teal green palette`。

### 方向 B：夜塘月色（Moonlit Pond / Cinestill 夜潭）

**气质**：Cinestill 800T 电影夜——冷青蓝水 + 钨丝暖橙球。"胶片感"论据最硬（有真实胶片成像机理背书：冷青世界 + 点光源红橙 halation），离现在的"夜"最近、风险最小。

- **背景**：`#06141d`，渐变 `#020b15 → #06141d → #0c2733`，加一条窄垂直"月光水路" `#34606844 → transparent`（替代极光的视觉锚点）。
- **球色板**：`#E8A85C`（钨丝橙）、`#C8503C`（halation 红橙，限 2–3 球）、`#E8D8A8`、`#D89AA8`（荷粉）、`#3A8A8C`、`#5C9C94`、`#2C5878`、`#9ACFD4`（冰青）。
- **光色**：月光冷白青 `#E0F7FF` 替换所有纯白光元素；**播放中的球给 `#FF7A50` halation 暖晕**——Cinestill 红晕只出现在点光源上，语义恰好 = "正在发声的球"。
- **参考关键词**：`Cinestill 800T halation night`、`moonlight on water deep ocean eclipse`、`tungsten light reflection water`。

### 方向 C：萤光水塘（Bioluminescent Pond）

**气质**：最暗的水 + 最亮的点（Noctiluca 蓝绿萤光 + 萤火虫暖金双光源）。可点击显著性最强（亮点即交互点），视觉冲击最大，但离胶片感最远，需把饱和度压到 60–75% 防暗底振动。

- **背景**：`#020d0c`，中心微亮 `#052420`、四角 vignette `#010807`。
- **球色板**：`#41E0C4`、`#00B4A4`、`#6FC8E0`、`#8A86E0`（接续现有紫调）、`#F3D340`（萤火金）、`#E8A85C`（胶片记忆色保留位）、`#2E8A6E`、`#E8F4E0`。
- **光色**：主萤光 `#9AFFF7`，hover/播放瞬时 `#00FFE5`，暖侧 `#F8EE99`。
- **参考关键词**：`bioluminescent bay Vieques Noctiluca`、`fireflies over pond`、`ABZU dark zone glow`。

**三套共享的迁移主轴**：背景色相 240°→170°（明度不动）；白色光元素 → 月光冷白青/象牙；球保留 3–4 个暖胶片色做 teal-orange 前景层——显著性、胶片感、水塘感三者同时成立的公约数。

## 二、球体质感：池塘典型事物（纯 SVG 画法）

**底线质感（所有方向通用）**：零滤镜的三层 radialGradient 堆叠——①主体 `drop-body`（**中心透、边缘深**——和发光星球反着来，这是"水中物"的关键反转）②左上 10–11 点钟小椭圆主高光 ③6 点钟底部内反光细弱白弧（玻璃/水珠可信度的招牌细节）④可选 1px rim light。currentColor 单 def 复用 36 球，成本≈零。高光层 `pointer-events:none`，主体留事件。

**"池塘典型事物"的角色分配建议**（同一套渐变骨架，参数化变体）：
- **大球（importance 高）= 莲叶上的水珠 / 浮萍团**：边缘更实、高光更亮。
- **中球 = 漂浮的水泡（bubble）**：中心更透（fillOpacity 下探 0.35，但保 3:1 描边对比）、rim 更亮。
- **小球（深层）= 沉在浅水下的卵石/光斑**：更暗更绿、轻微 blur（复用 focus）——zMap 从"宇宙纵深"重新解释为**水深**：z 大 = 沉得深 = 更暗更绿更糊更晃。
- 可点击性铁律：球缘高光弧/描边对暗水保 ≥3:1；其他球播放时的淡出 opacity 不低于 0.35（或描边不淡）。

## 三、动效语言：从"飘在太空"到"浮在水上"

调研给出的三条**水感成立的关键原则**：

1. **涟漪有源、有寿命、会减速**——每个涟漪诞生于一个事件（点击、落叶入水、水黾点水、音符落水），`cubic-bezier(0.2,0.6,0.35,1)` 减速扩散后消亡，stroke-width 3px→0.5px 衰减；禁止无源永动同心圆。所有元素动作统一经 `bg-ripple:wave` 总线兑现成水面反应。
2. **多频叠加 + 全员异相**——每球 = 慢大涌（10–20s）× 快小漪（2–4s）× 噪声漂移三层叠加，36 球相位互不相同；任何两元素同步摆动立刻穿帮成机械感。
3. **受扰必回摆**——被涟漪推开的球要过冲→回摆→衰减（欠阻尼弹簧），而非均匀摩擦滑回。这是"浮在水上"区别于"飘在太空"的最直白物理签名。

**落地架构**（与 d3-force 零冲突）：
- d3 继续每 tick 写**外层** `<g>` 的 x/y；bobbing 用 CSS keyframe 挂**内层** `<g>`（translateY 0→-4~8px→0，3–6s + 双轴不可通约周期 + ±1.5° 微旋，hash 错峰 delay）——不同元素不同坐标系，CSS 在合成线程跑，不占 rAF 预算。
- d3 参数：**削弱 forceManyBody 和 forceLink（去星系结构感）**，调高 velocityDecay（更黏滞像水），anchor 改弹簧式（欠阻尼回摆），新增噪声微流 force。
- **弱化 3D 感**：perspective 默认关或大幅减幅；tilt 减幅；focus 重释为水深模糊（保留）。

**音乐 × 水（无频谱也能做）**：点击播放 = "一滴音落水"——球心发一组更亮更大的涟漪 + 广播 `bg-ripple:wave` 推开邻球；播放中按假节拍（1.2–2s 周期）持续发周期涟漪 + displacement scale 轻微脉动；暂停/曲终 = 涟漪绽放一次然后平息。播放指示从"日食月亮"改为**水中月倒影**：播放球附近一轮被 feTurbulence 轻扭、随波碎裂复圆的月影（改造现 EclipseLayer）。

## 五、P8-A 技术纪律清单（feTurbulence + feDisplacementMap）

```
type=fractalNoise（扭曲柔和不撕裂）  baseFrequency=0.012 0.018（静止！）
numOctaves≤2（≥5 视觉无差纯烧 GPU）  scale 基线 14（≈±7px，对 14–28px 球刚好）
region 显式收紧 x/y/width/height      位移后加 feGaussianBlur 0.4 修边缘锯齿
```
- 动画化：✅ 静噪声 + filter 内层 `<g>` 缓慢平移（球穿过噪声场自然晃动）；✅ rAF 事件脉冲动 scale；❌ 逐帧动 baseFrequency（每帧重生成整张噪声图）；❌ 动 seed（离散闪跳）。
- 挂载：按 z 分 2–3 组 `<g>` 各挂一份共享 filter，不同组不同 scale = "水深=晃动幅度"绑 zMap，一石二鸟。
- 降级：移除 filter attribute（scale=0 部分浏览器仍计费）；移动端默认关（既有策略正确，SVG filter 移动端 GPU 加速碎片化是社区一致结论）。
